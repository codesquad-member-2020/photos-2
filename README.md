# 브랜치 관리

- `master`: 온전한 개발 사항이 반영된 브랜치로 dev에서 merge함
- `dev`: PR 보내는 브랜치.
- `feature/****`: 각 기능 담당 브랜치. dev로 PR 보냄.  step별로 PR.
- `feature/****/issue-number`: 각 step 밑에 issue 번호로 브랜치 만들어서 구현
